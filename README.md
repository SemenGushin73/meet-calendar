# MeetCalendar

MeetCalendar — backend‑проект на Java / Spring Boot для бронирования переговорных комнат с недельным календарным представлением.

---

## Возможности

* Аутентификация пользователей (Spring Security, HTTP session)
* Управление бронированиями переговорных комнат
* Недельный календарь бронирований
* Защита от пересекающихся бронирований
* Ролевая модель (USER / ADMIN)
* Единый формат ошибок REST API
* Полная Swagger / OpenAPI документация

---

## Технологический стек

* **Java** 25
* **Spring Boot 4**
* **Spring Web (REST)**
* **Spring Security** (session‑based)
* **Spring Data JPA**
* **Hibernate**
* **PostgreSQL**
* **Flyway** (миграции БД)
* **MapStruct** (маппинг DTO)
* **Swagger / OpenAPI 3**
* **Lombok**

---

## Архитектура

Проект построен по слоистой архитектуре:

```
web (REST controllers)
 └─ api handler (facade)
     └─ service (business logic)
         └─ repository (JPA)
```

### Основные принципы

* REST‑контроллеры **тонкие** — только HTTP и Swagger
* Вся бизнес‑логика находится в `service`
* REST‑контракты вынесены в `*Contract` интерфейсы
* DTO строго разделены:

    * API DTO (`web.api.dto`)
    * Service / UI DTO (`web.dto`)
* Централизованный `RestExceptionHandler`

---

## Основные доменные сущности

* `User` — пользователь системы
* `Role` — роль пользователя (USER / ADMIN)
* `Room` — переговорная комната
* `Booking` — бронирование комнаты

---

## REST API

Базовый URL:

```
/api/v1
```

### Бронирования

* `POST /api/v1/bookings` — создать бронирование
* `DELETE /api/v1/bookings/{id}` — отменить бронирование

### Календарь

* `GET /api/v1/calendar/week?week=yyyy-MM-dd`

    * если параметр `week` не передан — используется текущая неделя

## Обработка ошибок

Все ошибки REST API возвращаются в едином формате:

```json
{
  "timestamp": "2026-01-13T12:00:00+04:00",
  "status": 400,
  "error": "BOOKING_VALIDATION",
  "message": "Incorrect input data",
  "path": "/api/v1/bookings"
}
```

Используется `@RestControllerAdvice`.

## Валидация и бизнес‑правила

* Минимальная длительность бронирования: **30 минут**
* Максимальная длительность: **24 часа**
* Запрещены пересекающиеся бронирования
* Запрещено отменять завершённые бронирования
* Отменять бронирование может:

    * владелец
    * администратор

## Запуск проекта

1. Поднять PostgreSQL (Docker или локально)
2. Прописать настройки в `application.yml`
3. Применить Flyway‑миграции
4. Запустить приложение:

```bash
./gradlew bootRun
```
